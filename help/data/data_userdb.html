Numbrz Database

Numbrz provides a simple data hosting capability via the Numbrz database. You can use these hosted tables when you want to store some of your data within the Numbrz application. This can be very useful for interstitial, persistent data that contributes towards a final analysis but is not, itself, part of the ultimate results. For example, you may have a series of related models that leverage common data.

Configuring a Numbrz Database

Overview section
This section allows you to set a name and description for this database.

Tables section
This section allows you to manage the tables in the database. You can edit existing tables, add new tables, or delete tables. When adding columns to a Numbrz Table, note that the type of data must be specified. These types are slightly different than the ones encountered when working with Flows.

  • String - Use this type for a text value.
  • Integer - Use this type for a whole number without fractional value.
  • Decimal - Use this type for general numeric values and currency.
  • Date - Use this type for a date with no time (e.g.: '2/14/2020').
  • Date and Time - Use this type for a date with time (e.g.: '2/14/2020 10:45:30').
  • Boolean - Use this type for a true/false value.


Note: At this time, you cannot view or enter data directly into your Numbrz tables. Instead, prepare Flows and Jobs to populate the tables with data or to copy table data to an external target, for example, a spreadsheet.
