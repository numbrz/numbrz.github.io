Testing a Stage

Flow Stages perform the business logic of Models. They process sets of records by passing them through a sequence of Steps, which configure various transformations and calculations. When editing stages, it is useful to be able to confirm their proper operation, without having to set up and configure the entire flow, job, and data sources. The stage testing feature is designed to provide in-editor testing and trouble-shooting.

Making Stages Testable

Testing a stage starts with instrumenting all stage inputs with a small set of sample data. You must add sample data to the stage's Input section, as well as the Data section of all Combine steps defined in the stage. A button labeled Test Data is displayed in these sections in the upper right corner. Clicking this button will allow you to set up a few records of sample data. This sample data should include known test case values, such that the output of a test can be used to validate the stage's logic.

Running a Test

Running the test is simply a matter of clicking the Run Test button that appears in the upper right corner of stage settings once test input is defined. The test inputs will be passed through the stage, and for each step, the data values output by that step will be displayed for review.
