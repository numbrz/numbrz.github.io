Output Step

This step specifies what data fields and when in a sequence of steps, output to some target occurs. Multiple output steps can appear in a stage in any location within the sequence of steps. Records received as input to an Output step are always passed on to the following steps, regardless of configuration options. Most frequently, output will occur at the end of processing for any given stage.

Configuration

To configure an Output step, begin by first selecting a target.

Output to Job Configuration

Select this option when you want to the records go to a specific data target configured in a Job. The following attributes should then be completed.

  • Label - A friendly name that will make mapping this output more identifiable in Jobs.
  • Erase... - Check this to erase data in the target before writing - this setting can be overridden in Jobs.
  • Skip... - Check this to skip output (i.e. do no writing) - this setting can be overridden in Jobs.

Click Add Fields to select the data that will be output.

Optionally click Add Filter to configure a set of conditions to apply to records before output. Only records that match these conditions will be written. Configuring a filter here is similar to configuring a Filter Step.

Output to Working Table

Select this option when the output should be written to an in-memory working table. Working tables allow stages to share data and produce an analysis incrementally. The following attributes should then be completed.

  • Erase... - Check this to erase data in the working table before writing.
  • Select Working Table - Choose the working table to output to.
  • Map Fields - Map data fields from the record to the fields of the working table.

Optionally click Add Filter to configure a set of conditions to apply to records before output. Only records that match these conditions will be written. Configuring a filter here is similar to configuring a Filter Step.
