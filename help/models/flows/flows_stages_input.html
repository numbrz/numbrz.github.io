Stage Input

Flow Stages are not bound to specific data stored in any particular data source, but rather specify the general structure of the data they require as input. When configuring a stage, the first set of attributes to consider is this data structure specification, which is the top section of a stage configuration. There are two options for the input data to a stage.

Input from Job Configuration

Select this option when you want to connect the flow and its stages to a specific data source in a Job.

You can enter the data fields required by this stage by entering them in the Input Fields section. The IMPORT button will allow you to select a data source you have previously configured and copy the column structure from that source. This is a valuable short-cut if you know ahead of time that a specific data source has the fields the flow stage needs.

Input from Working Table

Select this option when the input of a stage is coming from an in-memory working table that has been populated by a previous stage. Working tables allow stages to share data and produce an analysis incrementally. If you select this option, you will also have the opportunity to configure an optional Filter to select a sub-set of records from the table.

Test Data

Once one or more data fields have been defined, you will see a button labeled Test Data in the upper right corner of the Input section. Clicking this button will allow you to set up a few records of sample data, which you can use to test your sequence of steps while editing a flow stage.
